# Contributing to FusionX-OS

## Scope
FusionX-OS is focused on modular ARM-first system architecture. Contributions should preserve clean boundaries between subsystems.

## How to Contribute
1. Open an issue or proposal describing scope and ownership impact.
2. Keep changes scoped to the owning subsystem directory.
3. Update relevant documentation in `/docs`.
4. Validate any new automation in `/scripts`.

## Documentation
- Architecture changes must update `docs/architecture`.
- Control plane changes must update `docs/control-plane`.

## Review Expectations
- Changes that affect subsystem interfaces require cross-team review.
